# Как найти Root Directory в Railway

## Вариант 1: Через настройки сервиса (новый интерфейс)

1. **Откройте ваш проект в Railway**
2. **Нажмите на название вашего сервиса** (не на проект, а на сам сервис - обычно это прямоугольник с названием)
3. **В открывшемся окне найдите вкладку "Settings"** (или иконку шестеренки ⚙️)
4. **Прокрутите вниз** - там должно быть поле **"Root Directory"** или **"Working Directory"**

## Вариант 2: Через Deployments

1. Откройте ваш проект
2. Перейдите во вкладку **"Deployments"**
3. Нажмите на три точки (⋮) рядом с последним деплоем
4. Выберите **"View Settings"** или **"Configure"**
5. Там должно быть поле Root Directory

## Вариант 3: Через Variables (альтернативный способ)

Если Root Directory недоступен, можно использовать переменную окружения:

1. Перейдите в **Settings** → **Variables**
2. Добавьте новую переменную:
   - **Name:** `RAILWAY_WORKING_DIRECTORY`
   - **Value:** `shop_project`
3. Сохраните

## Вариант 4: Через railway.json (если поддерживается)

Попробуйте добавить в `railway.json`:

```json
{
  "$schema": "https://railway.app/railway.schema.json",
  "build": {
    "builder": "NIXPACKS",
    "workingDirectory": "shop_project"
  },
  "deploy": {
    "restartPolicyType": "ON_FAILURE",
    "restartPolicyMaxRetries": 10
  }
}
```

## Вариант 5: Переместить файлы в корень (простое решение)

Если настройка Root Directory недоступна, можно переместить ключевые файлы Django в корень репозитория. Но это более сложный вариант.

## Что делать прямо сейчас:

1. **Сделайте скриншот** интерфейса Railway, где вы находитесь
2. Или опишите, какие вкладки/разделы вы видите в Settings
3. Попробуйте найти любое поле с названием, содержащим "Directory", "Path", "Working" или "Root"

## Альтернативное решение: Использовать Render.com

Если Railway продолжает вызывать проблемы, Render.com проще настраивается:
- Автоматически определяет подпапки
- Проще интерфейс
- Подробная инструкция в `DEPLOY.md`
